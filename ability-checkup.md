The official text is the resolved `ABILITIES` description after `backend/game/abilities/ability-property-overrides.json` is applied by `buildResolvedAbilities()`. Runtime was checked against `backend/game/abilities/abilities.ts`, `backend/game/engine/flow/play/immediateEffects.ts`, `backend/game/engine/flow/play/PlayAbility.ts`, `backend/game/engine/flow/play/buffs.ts`, `backend/game/services/gameplay/playService.ts`, `backend/game/engine/loop/GameLoop.ts`, `backend/game/engine/effects/buffRuntime.ts`, `backend/game/engine/effects/onDamageHooks.ts`, `backend/game/engine/effects/definitions/DirectionalDash.ts`, and `backend/game/abilities/buff-attribute-overrides.json`.

## Cross-Cutting Runtime Details

| Topic | Runtime detail | Why it matters | Where checked |
|---|---|---|---|
| Buff editor overrides | `addBuff()` applies `buff-attribute-overrides.json` live through `applyPropertyOverridesToEffects()`. This changed actual runtime for `韦陀献杵` self buff and `惊鸿游龙` in this audit. | Ability text can look correct while buff-editor properties silently change applied effects. | `buffRuntime.ts`; `buffEditorOverrides.ts`; `buff-attribute-overrides.json` |
| Control diminishing returns | Any hostile `CONTROL`, `ROOT`, freeze, or shared lockout buff applied through `addBuff()` can have duration shortened by resistance stacks. | Official descriptions list base durations; actual durations may be reduced after prior controls. | `buffRuntime.ts` `getResistanceConfig()` / `refreshResistanceBuff()` |
| 盾立 reflection whitelist | Some abilities are explicitly `dunLiWhitelisted`; reflected damage/effects are not universally "all skills" despite 盾立 text. | Text says all enemy skills reflect, but runtime has per-ability exceptions. | `ability-property-overrides.json`; `dunLiReflect.ts`; `PlayAbility.ts`; `buffRuntime.ts` |
| Projectile immunity | `PROJECTILE_IMMUNE` blocks damage/buffs/effects from abilities marked `isProjectile`. | Official descriptions often do not mention projectile flags, but they materially change whether skills land. | `ability-property-overrides.json`; `Damage.ts`; `immediateEffects.ts`; `buffs.ts`; `GameLoop.ts` |
| Area target caps | Many descriptions say `最多N`, but many runtime loops use all hostile targets found in range unless the effect explicitly has `maxTargets` and the handler reads it. | This affects multi-player modes and entity targets heavily. | `immediateEffects.ts`; `GameLoop.ts` zone/channel loops |
